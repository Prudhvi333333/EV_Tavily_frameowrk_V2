End-to-End Flow

Input
Questions + KB
  • Load split from Excel (train/test)
  • Load KB master rows
  • Build/attach Chroma index
Retrieval
Hybrid + LlamaIndex
  • Query intent detection
  • Semantic (LlamaIndex over Chroma)
  • BM25 lexical blend
  • Metadata boost (tier/location/role)
Rerank
Cross-Encoder
  • KB reranker: cross-encoder/ms-marco-MiniLM-L6-v2
  • Optional web rerank over accepted web docs
Generation
Qwen Pipelines
  • rag, no_rag, rag_pretrained, rag_pretrained_web
  • Few-shot + CoT prompting controls
  • HyDE optional expansion
Evaluation + UI
Judge + Reports
  • Metric scoring + validator
  • Excel outputs + comparison report
  • Dash reviewer UI + proof log

Tavily Metadata Filtering (Production Policy)

Now Implemented
  • Pre-scrape policy filter on Tavily candidates
  • Registry-driven domain blocklist from Rejected_Documents / Failed_Acquisitions
  • Allowlist override from Review_Ready.Final_Decision
  • Score thresholds: metadata and credibility
  • Policy rejections are logged in proof records
Data Dependency
  • Registry file: rag_data_management_registry.xlsx
  • If policy is enabled and registry is missing, run fails fast
  • Config: crawler.metadata_filtering.*

This filter runs before Firecrawl scrape, reducing wasted crawl calls on known-poor domains.
